# ai-employee-kit v0.2.0 — Path B Modernization

## Context

`ai-employee-kit` v0.1.0 shipped an `INSTALL-GUIDE.md` Option C that instructed adopters to copy 8 harness scripts from `bin/` into `~/.claude/bin/`. The repo actually shipped 2 scripts (`check-contamination.sh`, `denylist.txt`). Option C was a broken install path. The `README.md` "What's In the Kit" table mirrored the same un-shipped references.

Path B (operator-approved in ai-governance-standards C7, chain G20260418-062502-300-C7): remove the harness-install surface from the kit entirely and point adopters upstream to `strategicthings/ai-governance-standards` for the hooks. The kit focuses on what it actually provides: the governance protocol doc, ABOUT template, quick reference, and the standalone contamination scanner.

This is a breaking change for v0.1.0 users who attempted Option C. CHANGELOG documents the migration path.
